# Custom Types
Many CeTZ functions expect data in certain formats which we will call types. Note that these are actually made up of Typst primitives.

## <Type>coordinate</Type>
A position on the canvas specified by any coordinate system. See [Coordinate Systems](coordinate-systems).

## <Type>number</Type>
Any of <Type>float</Type>, <Type>int</Type> or <Type>length</Type>.

## <Type>style</Type>
Named arguments (or a dictionary if used for a single argument) of style key-values.

## <Type>context</Type>
A CeTZ context object that holds internal state.

## <Type>vector</Type>
A three element <Type>array</Type> of <Type>float</Type>s.
